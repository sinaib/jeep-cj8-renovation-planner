💾  Prompt Caching

The CJ8 knowledge + agent rules (static system prompt, ~70% of tokens) is marked cache_control: ephemeral. After the first request per session, Anthropic caches it — subsequent hits cost ~10% of normal. The dynamic part (live plan) is rebuilt fresh each time.

🗜️  Conversation Compression

After 12+ messages in the conversation, the oldest 8 are summarized into a single compact note via Haiku. The 4 most recent messages stay raw. Nothing meaningful is lost — the full plan is always in the system prompt. Logged as history_compressed.

Every tool maps directly to a Zustand action or an external call. The agent picks the right ones, calls them in sequence, and loops until the work is done — without you having to ask step by step.

⚡ Task Auto-Population Rule

Whenever the agent calls add_task, the system prompt mandates it immediately follow with:

  1. set_task_steps — 3–6 CJ8-specific steps with torque specs and known gotchas for this exact task
  2. add_part_to_task × N — every part needed, with Israeli market cost estimates and part numbers where known
  3. update_task_cost — total estimated cost in ₪

This means tasks are never just names — they're living documents from the moment they're created.

Efficiency
Smart Context Selection
The agent doesn't get the whole plan on every message. A selector in contextSelector.ts analyzes the query and picks only the relevant slice — reducing tokens by 50–80% on most requests while keeping answers accurate.
Context Selection Logic → buildDynamicContext(query, opts)
🎯 Task in focus
opts.taskId provided
Focused Task Detail — Full task: name, status, priority, parts list, steps, all notes, cost, blockers. Plus sibling tasks in the same phase (compressed). Used when TaskDetailView is open.
📦 Phase in focus
opts.phaseId only
Phase View — All tasks in that phase, each with status and dependencies. Completion percentage. Used when browsing a specific phase.
🔧 System keyword
"engine", "brakes", "axle", "steering", "electrical"…
System-Filtered Tasks — Only tasks where systemId or task name matches the detected vehicle system. Covers 12 systems (engine, fuel, cooling, transmission, driveshafts, brakes, steering, suspension, electrical, body, frame, interior).
💰 Cost keyword
"cost", "budget", "price", "₪", "money"
Cost Summary — Total estimated vs. actual spend by phase. Individual task costs. Useful for budget reviews.
📋 Default
no match above
Compressed Plan — Phase names + progress (e.g. "3/8 done, 2 active"). Task names + statuses + priorities + costs. No steps, notes, or parts. Enough for general status questions.
Always included regardless of mode:  Car profile · Recent decisions (last 5) · Active gaps · All phase and task IDs (so agent can reference anything by ID).
Research notes included only if topic matches the query or the focused task.
50–80%

Fewer tokens per request compared to always sending the full plan. On a 40-task plan, a focused task query sends ~500 tokens instead of ~4,000.

📸 Prompt Caching — how it splits
Static (~70% of tokens) — CJ8 knowledge, agent rules, approach, communication style. Cached at ~10% cost after first request.
Dynamic (~30% of tokens) — Live plan state, selected by context mode. Rebuilt every request.

3 Background Intelligence Triggers
The agent doesn't wait to be asked. Three triggers fire automatically at different moments — scanning for gaps, populating newly unblocked tasks, and running a weekly health check. All silent: no UI callbacks, no error popups.
🔄
On Plan Change
Gap Scanner
Trigger
Any plan change — debounced 8s, max once per 10 minutes
What it does
Scans for CJ8-specific failure points with no corresponding task, missing safety-critical items, and logically required dependencies not yet captured
Tools used
flag_gap
🔇 Fully silent
✅
On Task Completion
Unblock Analysis
Trigger
Task marked 'done' — only fires if it unblocks dependent tasks. 3s delay to let store settle.
What it does
For each newly-unblocked task: checks if parts + steps are populated, fills them in if not. Adds a note about what to watch out for based on what was just completed.
Tools used
set_task_steps add_part_to_task add_task_note
🔇 Fully silent
📅
Weekly Health Check
Plan Hygiene
Trigger
App load — at most once per week. Timestamp tracked in localStorage.
What it does
Flags "active" tasks with no recent notes (stale/forgotten). Flags phases with >10 tasks (too big, consider splitting). Flags missing restoration phases (no interior? no safety inspection?).
Tools used
flag_gap add_task_note
🔇 Fully silent
Data
Where Everything Lives
Data is persisted to disk via a Vite dev server plugin that adds 5 Node.js API routes. No separate backend needed — the Vite process is the server. localStorage acts as a sync backup; IndexedDB holds binary files.
💾 Disk Files via Vite server plugin (Node fs)
data/
  project.json         ← full Zustand state
  changelog.ndjson    ← append-only event log
  snapshots/
    2026-03-14T15-52.json
    ↑ auto-saved before bulk agent changes
  files/              ← .gitignore'd
Write strategy: localStorage written synchronously (instant), then disk written async. On load: disk is tried first, localStorage is fallback.
