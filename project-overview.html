01 — PURPOSE

What is this?

Not a generic project tracker. A specialist tool, hand-built for one specific mission.

🎯 The Mission

Track the complete restoration of a 1989 Jeep CJ8 Scrambler — from fully disassembled (engine out, tub off frame, wiring stripped) to a clean, trail-capable daily driver. The app keeps all plans, decisions, costs, and work notes in one living document.

🔧 The Problem It Solves

Restoration projects have hundreds of interdependent tasks spread over years. Parts go missing. Decisions get forgotten. Costs spiral. This app is the single source of truth — with a built-in AI advisor who knows the CJ8 platform inside-out and helps plan the work.

💾 Why Local-First?

No account. No cloud. No subscription. The app runs on your own machine at localhost:5175. All data saves directly to project files on disk — the JSON files are yours, readable, git-trackable, and will never disappear.

02 — HOW IT WORKS

The overall flow

From first launch to ongoing restoration tracking — three modes, one seamless journey.

🛰️
PHASE 01

Onboarding

The AI advisor asks about your Jeep — condition, goals, tools, budget. It researches the CJ8 platform, records car facts, and starts building the plan live as you talk.

🗺️
PHASE 02

Plan Built

The advisor has created phases and tasks. You review them, ask questions, let it expand tasks with steps and parts lists. Approve when it looks right.

🔩
PHASE 03

In Progress

You're doing the work. Mark tasks active → done. Add notes and photos. The advisor stays available for technical questions, part sourcing, and plan adjustments.

📖
PHASE 04

Journey

Every completed task becomes a permanent, searchable log entry with photos, costs, notes, and dates. The full history of your restoration — auto-built as you work.

Key idea: The AI advisor isn't a chatbot that gives generic advice. It has tools — it can directly edit the plan, add tasks, set dependencies, flag risks, save research notes, and update costs. Everything it says gets written into your data, not lost in chat history.

04 — THE AI ADVISOR

How the AI works

Not a chatbot — an agent. It thinks in a loop, uses tools, and directly edits your plan.

The Agentic Loop

When you send a message, the advisor doesn't just reply with text. It thinks, then calls tools to make changes, then thinks again, then calls more tools — until there's nothing left to do. Only then does it respond.

This means asking "what should I check first?" might cause it to: research the CJ8 engine, add 3 new tasks, set their dependencies, update costs, and flag a gap — all silently, before answering you.

Auto-snapshot: If the advisor makes 2+ changes in one turn (a bulk change), the app automatically saves a full snapshot of your plan before applying the changes — so you can always roll back.

The 23 Tools

Everything the advisor can do — grouped by category.

🔍 Research
search_web(query)
DuckDuckGo search. Used to look up CJ8 specs, parts, known failures, prices, technical data.
add_research_note(topic, finding)
Saves a research insight permanently to the store — linked to relevant tasks.
set_car_fact(key, value)
Records a confirmed fact about your specific Jeep (e.g. engine_condition = unknown, not run in 8yrs).
record_decision(category, summary)
Logs a project decision with rationale — preserved forever in the Journey and decision log.
📐 Plan Structure
add_phase(name, subtitle, systemIds, order)
Creates a new phase (e.g. "Engine & Drivetrain"). Phases group related tasks and appear in the journey strip.
add_task(name, phaseId, priority, cost)
Creates a task with priority, phase, and optional cost estimate. Always followed by set_task_steps and add_part_to_task.
set_task_dependency(taskId, dependsOnTaskId)
Declares that task A cannot start until task B is done. Shown in the WORK NOW view.
get_full_plan()
Reads the entire current plan — all phases, tasks, statuses, costs. Used to reason about what's missing.
🔩 Task Enrichment
set_task_steps(taskId, steps[], guide?)
Saves a step-by-step technical guide for a task. The steps appear in the task detail view.
add_part_to_task(taskId, partName, cost)
Adds a required part/material with estimated cost. Parts list appears in task detail with purchase checkboxes.
update_task_cost(taskId, costILS)
Sets the estimated cost of a task in Israeli Shekels. Totals roll up to phase and overall budget.
add_task_note(taskId, note)
Appends a timestamped note to a task. Notes appear in the task detail and in the Journey log.
⚠️ Risk & Management
flag_gap(systemId, description, severity)
Flags a risk or missing item (critical/warning/suggestion). Shows as a banner until dismissed or converted to a task.
update_task_status(taskId, status)
Changes a task status: flagged → todo → active → done → skipped.
remove_task / move_task
Delete a task from the plan, or move it to a different phase. Used when restructuring the plan.
annotate_file(fileId, note)
AI saves an observation on an uploaded photo or PDF — e.g. "frame rail shows significant surface rust at rear crossmember".

What the system prompt tells the advisor

🚙 CJ8 Platform Knowledge

The system prompt includes detailed CJ8-specific technical context: AMC 150 2.5L engine quirks, Dana 30/300 specs, known failure points (rear main seal, AMC 20 axle shafts, head gaskets), and common restoration sequences.

📊 Your Live State

Every request rebuilds the system prompt with your current plan (all phases/tasks/IDs), car facts, decision log, research notes, and gaps. The advisor always has full context — it never forgets what's been built.

🎯 Operating Rules

The advisor is instructed to: research before advising, ask one question at a time, price in Israeli Shekels, use tools silently, always enrich new tasks immediately with steps + parts + costs.

How the Vite server becomes the backend: The app uses a configureServer hook in vite.config.ts to add Node.js file system middleware. This means the Vite dev server (npm run dev) serves both the React app AND acts as a tiny API — no separate backend, no Docker, no cloud. Just one command.
06 — THE DATA MODEL

What's in the store

Zustand manages all app state — the plan, intelligence layer, agent conversation, and UI. One object, always in sync.

🗺️ THE PLAN
  • phases — Phase[]
  • tasks — Record<id, Task>
  • gaps — Gap[]
  • taskDependencies — Dependency[]

  • Task includes:
  • name, systemId, phaseId
  • status todo→active→done
  • priority critical/high/medium/low
  • estimatedCostILS
  • actualCostILS
  • parts — Part[] with purchase state
  • steps — string[] guide
  • notes — timestamped string
  • manualRefs
  • completedAt
  • agentRationale
🧠 INTELLIGENCE
  • carFacts — CarFact[]
  • decisions — Decision[]
  • researchNotes — ResearchNote[]

  • CarFact:
  • key / value / confirmedBy
  • e.g. engine=AMC 150, condition=unknown

  • Decision:
  • category budget/approach/scope…
  • summary + rationale
  • madeBy user or agent

  • ResearchNote:
  • topic + finding + source
  • relevantTaskIds
📂 FILES + AGENT
  • fileIndex — FileMeta[]
  • agentHistory — AgentMessage[]

  • FileMeta:
  • taskId / phaseId link
  • name, type, size
  • caption — user label

  • AgentMessage:
  • role user / assistant
  • content — text
  • toolCalls — ToolCall[]

  • App:
  • appState onboarding/built/in_progress
